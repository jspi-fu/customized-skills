## 文本对齐
---
### 居中对齐

```markdown
<p align="center">居中显示的文本</p>
```

**渲染效果**：文本在页面水平中央显示

**应用场景**：

- 书籍序言/结语中的核心观点
- 重要声明或口号
- 章节引言
- 致谢词

---

### 居中加粗

```markdown
<p align="center"><strong>亲爱的同志们，不要成为我，要超越我。</strong></p>
```

**应用场景**：

- 书籍赠语
- 重要论断
- 需要强调的核心观点

---

### 居左对齐

Markdown 默认即为左对齐。如需在 HTML 块中显式指定：

```markdown
<div align="left">
左对齐的文本内容
</div>
```

## 标题与分割线
---
### 二级标题 + 分割线模式

```markdown

## 困惑与醒悟
---
正文内容...
```

**排版规则**：

1. 使用 `##` 定义二级标题
2. 标题下方紧跟 `---` 分割线
3. 分割线后空一行再开始正文

**应用场景**：

- 书籍章节划分
- 长文档的板块分隔
- 增强章节的视觉独立性

### 标题层级规范

```markdown
# 一级标题 - 页面主标题

## 二级标题 - 主要章节
---
### 三级标题 - 章节内小节

#### 四级标题 - 详细内容分组
```

## Admonition 提示框
---
### 基础语法

```markdown
!!! 类型 "可选标题"
    提示内容（必须缩进4个空格）
    
    支持多行内容
```

---

### 支持类型

| 类型 | 颜色 | 适用场景 |
|------|------|----------|
| `note` | 蓝色 | 一般性提示、注意事项 |
| `tip` | 绿色 | 技巧、建议、最佳实践 |
| `warning` | 黄色 | 警告、需要注意的问题 |
| `danger` | 红色 | 危险操作、重要警告 |
| `info` | 蓝色 | 信息说明、补充内容 |
| `question` | 紫色 | 问题、疑问解答 |

---

### 示例

```markdown
!!! note "提示"
    这是一个提示框的内容。

!!! warning "注意"
    这是一个警告框的内容。

!!! danger "危险"
    这是一个危险提示框的内容。
```

---

### 使用决策指南

| 类型 | 使用场景 | 示例 |
|------|----------|------|
| `!!! tip` | 技巧、建议、最佳实践、优化方案 | "使用 `uv run` 无需手动激活虚拟环境" |
| `!!! note` | 补充说明、额外信息、注意事项 | "UV 支持 Python 3.8+ 版本" |
| `!!! warning` | 警告、需要注意的问题、潜在风险 | "不要把 API 密钥提交到 Git" |
| `!!! danger` | 危险操作、会导致严重后果的行为 | "删除操作不可恢复" |
| `!!! info` | 一般性信息、参考链接、相关资源 | "更多信息请参考官方文档" |

**决策要点：**

- 技巧/建议 → `tip`（绿色，积极正面）
- 补充说明 → `note`（蓝色，中性信息）
- 需要注意 → `warning`（黄色，提醒注意）
- 严重后果 → `danger`（红色，必须警惕）

## Details 可折叠块
---
### 基础语法

```markdown
/// details | 点击展开查看详情
这里是详细内容，可以放代码、列表等任何 Markdown 内容。

- 支持列表
- 支持代码块
- 支持其他 Markdown 语法
///
```

**注意事项**：

- `///` 后必须有空格
- `details |` 中的 `|` 两侧必须有空格
- 结尾必须有 `///`

---

### 应用场景与决策指南

| 场景特征 | 示例 | 决策 |
|----------|------|------|
| 内容较长，但不是所有人都需要看 | 详细解释、背景知识、实现原理 | ✅ 使用 `/// details` |
| 问答形式的 FAQ | "什么是 XXX？"、"为什么要...？" | ✅ 使用 `/// details \| ❓ 问题标题` |
| 可选阅读内容 | 进阶用法、额外配置、参考资料 | ✅ 使用 `/// details` |
| 代码示例过多 | 多种实现方式的代码展示 | ✅ 使用 `/// details` 折叠 |
| 内容简短且重要 | 核心概念、关键步骤 | ❌ 直接展示，不要折叠 |
| 需要连续阅读 | 教程步骤、流程说明 | ❌ 直接展示，不要折叠 |

**标题命名建议：**

- FAQ 问题：`/// details | ❓ 为什么要使用 UV？`
- 详细说明：`/// details | 📖 点击查看详细配置`
- 代码示例：`/// details | 💻 完整代码示例`

## Tab 标签页
---
### 基础语法

```markdown
/// tab | 选项卡 A
这是选项卡 A 的内容
///

/// tab | 选项卡 B
这是选项卡 B 的内容
///
```

---

### 应用场景与决策指南

| 场景特征 | 示例 | 决策 |
|----------|------|------|
| 互斥的选项/方式 | 不同操作系统、不同工具、不同方法 | ✅ 使用 `/// tab` |
| 并行对比内容 | 工具对比、方案对比、版本对比 | ✅ 使用 `/// tab` |
| 分类整理的速查表 | 环境管理/包管理/项目管理 | ✅ 使用 `/// tab` |
| 内容有依赖关系 | 步骤1 → 步骤2 → 步骤3 | ❌ 使用顺序列表，不要 Tab |
| 内容需要同时参考 | 多个相关概念的解释 | ❌ 直接展示，不要 Tab |

**标签命名建议：**

- 按平台：`/// tab | Windows`、`/// tab | macOS`、`/// tab | Linux`
- 按方式：`/// tab | 方式一：原生管理`、`/// tab | 方式二：兼容模式`
- 按工具：`/// tab | UV vs pip`、`/// tab | UV vs Conda`
- 加 emoji：`/// tab | 🖥️ 环境管理`、`/// tab | 📦 包管理`

## 常见问题排查
---
### Q: Details/Tab 语法不生效

**检查项**：

1. 确认 `pymdownx.blocks.details` 和 `pymdownx.blocks.tab` 已添加到 `markdown_extensions`
2. 语法格式必须正确：`/// details | 标题`（注意空格）
3. 结尾必须有 `///`

---

### Q: Admonition 提示框不生效

**检查项**：

1. 确认 `admonition` 已添加到 `markdown_extensions`
2. 使用 `!!!` 三感叹号
3. 内容必须缩进4个空格

---

### Q: 数学公式不渲染

**检查项**：

1. 确认 `pymdownx.arithmatex` 已配置 `generic: true`
2. 检查公式语法是否正确
